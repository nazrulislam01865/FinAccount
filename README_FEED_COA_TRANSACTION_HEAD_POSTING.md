# Feed COA and Transaction Head Posting

## Final accounting structure

The Feed module uses the existing accounting transaction types. It does not create separate Feed transaction types.

- `PURCHASE` -> dedicated `Feed Purchase` transaction head (`SYS-FEED-PUR`)
- `SALE` -> dedicated `Feed Sale` transaction head (`SYS-FEED-SAL`)

The heads remain hidden from the generic Transaction Entry form because feed transactions must also update item, warehouse, batch, stock movement and weighted-average cost data.

## Head-specific accounting rules

Accounting rules can now optionally be scoped to a Transaction Head. Rule matching follows this order:

1. Active rule for the selected Transaction Head and payment type.
2. Active global rule for the Transaction Type and payment type.

The Feed module requires three active rules for each Feed head:

- Cash / paid in full
- Credit / fully due
- Partial / part paid

These rules are visible and manageable from **Accounting Rules**. The transaction-head scope, transaction type and payment type of system Feed rules are protected so the Feed mapping cannot be accidentally moved to another head. The rule name and active status can still be managed.

## Posting flow

### Feed Purchase

The Feed Purchase form posts through the `Feed Purchase` head under `PURCHASE`:

- Debit the COA connected to the Feed Purchase head (Feed Inventory).
- Credit the selected money-account COA for the paid amount.
- Credit the supplier payable COA for any due amount.

Transport and other allocated purchase costs are included in inventory value.

### Feed Sale

The Feed Sale form posts through the `Feed Sale` head under `SALE`:

- Debit the selected money-account COA for the received amount.
- Debit the customer receivable COA for any due amount.
- Credit the COA connected to the Feed Sale head (Feed Sales).
- Debit Feed COGS and credit Feed Inventory using weighted-average inventory cost.

The main sale/purchase journal is generated by the central accounting posting service and its head-specific rules. The Feed module appends the calculated COGS pair to the same sale journal.

## Feed Setup

The old manual Accounting Connection section is removed. Feed Setup now manages operational records only:

- Feed items
- Warehouses
- Default warehouse
- Batch and expiry behavior
- Default purchase and sale prices

The current Feed heads and their linked COA accounts are shown as an informational notice.

## Existing installations

Run the migration:

```bash
php artisan migrate --force
```

The migration adds `transaction_head_id` to `accounting_rules` and creates Cash, Credit and Partial head-specific rules for existing Feed Purchase and Feed Sale heads.

## Safety

- Generic Transaction Entry cannot select `SYS-FEED-*` heads.
- Feed posting fails clearly if a required head, linked COA, COGS account or head-specific rule is inactive or missing.
- Duplicate request tokens cannot connect a Feed document to a different accounting transaction.
- Accounting transaction, journal, feed document and stock changes remain in one database transaction.
