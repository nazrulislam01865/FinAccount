# HisebGhor Three-Level Chart of Accounts Update

## Implemented hierarchy

The Chart of Accounts now supports exactly three levels:

1. **Level 1 — Main Group**
   - Select `None / Main Parent` in the Parent Account field.
   - Example: `1000 Assets`.
2. **Level 2 — Category / Sub-group**
   - Select a Level 1 account as the parent.
   - Example: `1100 Current Assets`.
3. **Level 3 — Posting Ledger**
   - Select a Level 2 account as the parent.
   - Example: `1110 Cash in Hand`.

A Level 3 account cannot be selected as a parent, so a fourth level cannot be created.

## Automatic numbering

Codes are generated by the latest sequence of the selected level:

- Level 1: `1000`, `2000`, `3000`, ...
- Level 2 under `1000`: `1100`, `1200`, `1300`, ...
- Level 3 under `1100`: `1110`, `1120`, `1130`, ...

The backend regenerates the final code inside a locked database transaction, so two users cannot save the same code concurrently.

## Type inheritance

- Level 1 allows the user to choose the account type.
- Level 2 automatically inherits its Level 1 parent type.
- Level 3 automatically inherits its Level 2 parent type.
- A parent with child accounts cannot change parent or type until its children are moved or deleted.

## Posting safety

Only Level 3 accounts are available for:

- Money Account COA mapping
- Party receivable/payable mapping
- Transaction Head posting account mapping
- Due settlement account validation

An account already used by money accounts, parties, transaction heads, or journal lines cannot be changed into Level 1 or Level 2.

## Existing live data

The migration preserves all existing flat COA records as **unassigned Level 3 posting ledgers**. Existing transactions and mappings continue to use the same record IDs and codes.

An existing flat ledger can later be assigned to a Level 2 parent. Its hierarchy code is regenerated when it is moved.

## COA list improvements

The Chart of Accounts page now includes:

- Hierarchical row order and indentation
- Level 1, Level 2, and Level 3 badges
- Parent account column
- Level summary cards
- All Levels / Level 1 / Level 2 / Level 3 filter
- Search by code, account name, parent, account type, or level
- Child-count display
- Protection against deleting a parent that still has children

## Deployment

The production Vite build is included in `public/build`, so Node.js is not required on the server.

```bash
cd /var/www/hisebghor

php artisan down

git fetch origin main
git pull --ff-only origin main

sudo -u www-data php artisan optimize:clear
sudo -u www-data php artisan migrate --force

sudo chown -R www-data:www-data /var/www/hisebghor
sudo chmod -R 775 storage bootstrap/cache
sudo chmod -R 755 public/build

sudo -u www-data php artisan config:cache
sudo -u www-data php artisan route:cache
sudo -u www-data php artisan view:cache
sudo -u www-data php artisan queue:restart

sudo systemctl restart php8.4-fpm
sudo nginx -t
sudo systemctl reload nginx

php artisan up
```
