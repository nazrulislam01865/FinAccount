# Cash / Bank Setup Update

## Implemented

- Cash/Bank business ID is generated by the backend only.
- ID formats: `CB-00001` for Cash, `BK-00001` for Bank, and `MB-00001` for Mobile Banking.
- The ID is displayed read-only and is ignored if submitted or tampered with.
- Existing IDs and company ownership are immutable.
- ID generation is company-scoped and transaction-locked.
- Account names and account numbers are unique per company.
- Cash/Bank lists, dropdowns, linked ledgers, update, and delete actions are company-scoped.
- Linked ledger dropdowns are filtered by selected account type.
- Added `Mobile Wallet` ledger type for Mobile Banking accounts.
- Existing legacy Mobile Banking accounts linked to Bank ledgers remain editable without forced remapping.
- Type and linked COA cannot change after accounting history exists.
- Used Cash/Bank accounts cannot be deleted; they must be marked Inactive.
- Cash/Bank Book ignores deleted and cross-company setup rows.
- Editing unrelated fields no longer resets an existing opening balance to zero.

## Deployment

```bash
php artisan migrate
php artisan optimize:clear
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

Before deployment, back up the production database and `.env` file.
